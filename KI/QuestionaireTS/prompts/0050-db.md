Build a database layer for questionnaire management.

## Core Ideas

* Questionnaires have a title and a list of questions. Question types for now: `Text` (free-form string), `Boolean`, Likert scale
* For Likert scale:
  * `int` result, starting with 1
  * Configurable upper bound, min 3
  * Configurable label for lowest and highest value (e.g. _not at all familiar_ for lowest value, _expert_ for highest value)
* Answers for questions can be optional or required.
* Questionnaires are **versioned** — every update creates a new version with the full question list replaced (no partial updates).
* Deletion is **soft delete**.
* Users submit answers for a questionnaire. The full set of answers is posted in one request and validated against the questionnaire definition before storing.

## Tests

Write tests for the data access layer using an in-memory SQLite database.

1. **Create** a questionnaire "Customer Feedback Q2" with 4 questions (1x Text, 1x Boolean, 1x Likert 1-4, 1 Text optional, 1 Boolean optional).
2. **Update** it: Add a 6th question (Boolean, optional), tweak existing question text. This becomes version 2.
3. **Submit answers** for version 1 (5 answers).
4. **Submit answers** for version 2 (6 answers).
5. **List answers** and verify both sets are returned.

Add successful and failure cases (e.g. missing required answer, invalid question ID, submitting answers for non-existent questionnaire, etc.).

Remember:
* We use Drizzle as the ORM
* We use SQLite as our DB
* Entire data access logic must be in `packages/lib`, not in Next.js app. This includes code related to DB migrations.
